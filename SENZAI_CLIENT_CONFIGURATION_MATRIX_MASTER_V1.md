# SENZAI_CLIENT_CONFIGURATION_MATRIX_MASTER_V1

## Status

Universal Intake Standard — New Client Implementation Matrix
Owner: Senzai
Purpose:

This is the official discovery + configuration framework to use for every new Senzai client.

It is designed to transform raw client conversations into structured implementation intelligence so each new project can be built coherently and aligned with all official Senzai standards.

Use this matrix before architecture, coding, integrations, pricing finalization, or automation deployment.

This matrix becomes the source document for creating:

- CLIENT_CONFIGURATION_MATRIX_V1.md
- CLIENT_SYSTEM_ARCHITECTURE_V1.md
- CLIENT_SEO_MASTERPLAN.md
- CLIENT_AUTOMATION_PLAN.md
- CLIENT_SECURITY_MATRIX.md
- CLIENT_METRICS_PLAN.md

---

# CORE RULE

Every new client must be translated from chaos into systems.

Do not start building from assumptions.

Use this matrix first.

---

# HOW TO USE

For each new client:

1. Ask questions block by block.
2. Capture exact business reality.
3. Identify pain points.
4. Identify source-of-truth systems.
5. Identify revenue mechanics.
6. Identify growth opportunities.
7. Convert answers into custom client matrix.
8. Then begin architecture.

---

# BLOCK A — CLIENT IDENTITY

Purpose: Understand who the client is.

Questions:

1. Official business name?
2. Brand/public name if different?
3. Industry / niche?
4. Primary city / state / country?
5. Additional markets served?
6. Timezone?
7. Currency?
8. Primary language?
9. Secondary language(s)?
10. Single location or multiple locations?
11. Team size?
12. Owner/operator structure?

Output:

- tenant identity
- geo profile
- language model
- market scope

---

# BLOCK B — BUSINESS MODEL

Purpose: Understand how money is made.

Questions:

2. Secondary services?
3. High-ticket / low-ticket / recurring?

9. Refund policy?

11. Memberships/subscriptions?

# INTERVIEW RULES FOR CHATGPT

When using this matrix with a new client:

1. Ask only one block at a time.
2. Keep questions concise.
3. Push for exact numbers when possible.
4. Detect contradictions.
5. Identify hidden revenue leaks.
6. Recommend improvements during intake.
7. Think like CTO + CMO + COO + Growth Advisor.
